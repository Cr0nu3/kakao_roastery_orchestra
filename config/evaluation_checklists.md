# Evaluation Checklists (검증 체크리스트)

Evaluator가 문제 유형별로 사용하는 검증 체크리스트.
문제별 특수 조건은 `problem/description.md`, `problem/questions.md`에서 직접 읽어라.

---

## Universal Checklist (모든 문제 공통)

### 형식 검증
- [ ] 답안 형식이 문제 요구사항과 일치하는가
- [ ] JSON 답안의 경우 파싱 가능한가
- [ ] 필수 필드가 모두 존재하는가
- [ ] 값 범위가 허용 범위 내인가 (대소문자, 문자 종류, 숫자 범위)
- [ ] 배열 길이가 요구 사항과 일치하는가
- [ ] ID 중복이 없는가
- [ ] 글자수 제한이 있다면 준수되었는가

### 완전성 검증
- [ ] 모든 소문제에 답안이 있는가
- [ ] 빈 답안이나 placeholder가 남아있지 않은가

### Evidence 검증
- [ ] 코드로 계산된 답안에 evidence 파일이 존재하는가
  → evidence 없으면 해당 소문제 자동 FAIL

---

## TYPE_A: 데이터 분석형 Checklist

### 데이터 처리 검증
- [ ] 문제에서 요구한 전체 데이터가 처리되었는가 (필터링으로 데이터 누락 없음)
- [ ] 집계 단위(행, 그룹, 시간 범위)가 문제 조건과 일치하는가
- [ ] 날짜/기간 조건이 정확히 적용되었는가
- [ ] 수치 계산에 반올림/버림 기준이 명시된 대로 적용되었는가

### 통계/분석 검증
- [ ] 집계 함수(sum, mean, count, max, min)가 올바르게 사용되었는가
- [ ] 그룹화 기준이 문제 조건과 일치하는가
- [ ] 분류/예측 모델의 경우 훈련/검증 분할에 데이터 누수가 없는가

### 재현성 검증
- [ ] evidence 코드를 재실행하면 동일한 결과가 나오는가
- [ ] 하드코딩된 경로/값이 없어 다른 환경에서도 실행 가능한가

---

## TYPE_B: 코드 해석/실행형 Checklist

### 인터프리터 정확성 검증
- [ ] 언어 명세의 모든 연산자/명령이 정확히 구현되었는가
- [ ] 엣지 케이스(0 처리, 정수 나눗셈, 소수점 처리)가 명세에 따라 처리되었는가
- [ ] 함수/서브루틴 호출 스택이 올바르게 관리되는가
- [ ] 반복문의 종료 조건이 정확히 구현되었는가

### 실행 결과 검증
- [ ] stdin 입력값이 문제 조건에 맞게 설정되었는가
- [ ] 출력 형식(대소문자, 구분자 등)이 문제 요구사항과 일치하는가
- [ ] 동일 코드 재실행 시 결과가 일치하는가

---

## TYPE_C: 멀티미디어 추출형 Checklist

### 추출 완전성 검증
- [ ] 모든 미디어 파일이 처리되었는가 (파일 목록과 처리 목록 대조)
- [ ] 다중 추출 방법으로 교차 검증을 했는가 (OCR 단독 의존 방지)
- [ ] 특수문자/대소문자/공백 처리가 문제 요구사항과 일치하는가

### 추출 정확성 검증
- [ ] 무작위 샘플(최소 5건)을 직접 재확인했는가
- [ ] 추출 실패(빈 결과, 오류) 케이스가 없는가
- [ ] PDF의 경우 숨김 텍스트/레이어가 모두 탐색되었는가

---

## TYPE_D: 문서 종합 작성형 Checklist

### 사실 정확성 검증 (원본 역참조)
- [ ] 문서의 각 사실 주장이 원본 파일에서 확인 가능한가
- [ ] 원본과 모순되는 정보가 없는가
- [ ] 만료/무효 처리된 정보가 포함되지 않았는가 (문제에서 명시한 기준 시점 적용)

### 정보 완전성 검증
- [ ] 요구된 소스(이메일, 문서, 캘린더, 오디오 등) 각각의 핵심 정보가 반영되었는가
- [ ] 누락된 필수 항목이 없는가

### 포맷 준수 검증
- [ ] 제공된 템플릿의 구조(목차, 섹션 순서)를 정확히 따르는가
- [ ] 테이블 컬럼명, 리스트 형식이 템플릿과 일치하는가
- [ ] 문제에서 요구한 메타데이터(작성자, 날짜 등)가 정확히 설정되었는가

---

## TYPE_E: 공간 시뮬레이션형 Checklist

### 제약 조건 검증
- [ ] 차량/객체 사양(크기, 회전 반경 등)이 문제 명세와 일치하는가
- [ ] 조작 단위(이동, 회전 등)가 허용 범위 내인가
- [ ] 충돌 조건 검사가 모든 장애물에 대해 수행되었는가

### 목표 달성 검증
- [ ] 최종 상태가 목표 조건을 충족하는가 (위치, 각도 등)
- [ ] 제공된 시뮬레이터로 재실행했을 때 동일 결과가 나오는가
- [ ] 최적화 조건(최소 횟수, 최단 경로)이 요구된 경우 검토되었는가

## Verdict Decision Matrix

| CRITICAL 건수 | HIGH 건수 | Verdict |
|-------------|---------|---------|
| 0 | 0 | PASS |
| 0 | 1-2 | PARTIAL_PASS |
| 0 | 3+ | FAIL |
| 1+ | any | FAIL |
