# Problem Taxonomy (문제 분류 체계)

## 유형 분류 기준

문제 파일을 읽고 아래 유형 중 하나(또는 복합)로 분류한다.
분류는 Generator/Evaluator가 스스로 수행하며, 이 파일은 참조용이다.

---

### TYPE_A: 데이터 분석형

**특징**: 구조화된 데이터를 처리·분석하여 정량적 답을 도출
**판별 기준**: 대용량 JSON/CSV/TXT 데이터 파일 + 집계/분류/예측 질문
**핵심 도구**: Python (pandas, numpy, scikit-learn)
**검증 방법**: 코드 재실행 또는 독립 재계산

---

### TYPE_B: 코드 해석/실행형

**특징**: 비표준 언어(DSL) 또는 난독화된 코드를 해석·실행
**판별 기준**: 프로그래밍 언어 명세 또는 인터프리터 구현이 필요한 문제
**핵심 도구**: Python 실행 환경
**검증 방법**: 인터프리터 재구현 후 재실행

---

### TYPE_C: 멀티미디어 추출형

**특징**: 이미지/영상/PDF에서 정보를 추출
**판별 기준**: 텍스트 추출이 불가능한 미디어 파일 첨부
**핵심 도구**: pytesseract, pdfplumber, PyMuPDF, 영상 분석 라이브러리
**검증 방법**: 다른 추출 방법으로 교차 검증

---

### TYPE_D: 문서 종합 작성형

**특징**: 다중 소스 정보를 종합하여 구조화된 문서 생성
**판별 기준**: 복수의 이기종 파일(이메일/문서/오디오/스프레드시트)을 읽고 단일 문서로 통합
**핵심 도구**: 포맷별 파서 (docx, xlsx, m4a, pdf 등), 제공된 template
**검증 방법**: 원본 파일 역참조로 사실 확인

---

### TYPE_E: 공간 시뮬레이션형

**특징**: 물리 환경에서 최적 경로/조작 탐색
**판별 기준**: 좌표계 + 충돌 조건 + 시뮬레이터 파일 제공
**핵심 도구**: 제공된 simulator.py 또는 pygame 환경
**검증 방법**: 시뮬레이터 재실행으로 경로 유효성 확인

---

## 복합 유형

단일 문제가 두 유형 이상에 해당할 수 있다:

| 조합 | 예시 패턴 |
|------|---------|
| A+C  | 이미지에서 수치를 추출한 뒤 통계 분석 |
| B+C  | 이미지 속 코드를 OCR로 추출한 뒤 실행 |
| D+C  | 오디오/이미지 포함 다중 파일 문서화 |

복합 유형의 경우 각 유형의 검증 방법을 모두 적용한다.

---

## 유형 판별 흐름

```
problem/files/ 를 확인한다
│
├── 대용량 JSON/CSV/TXT 데이터? → TYPE_A
├── 언어 명세/DSL 파일?         → TYPE_B
├── 이미지/PDF/영상 파일?       → TYPE_C
├── 복수 이기종 파일 + 템플릿?  → TYPE_D
└── 시뮬레이터/좌표계 파일?     → TYPE_E
```

문제에 명시적 유형 힌트가 없으면 `description.md`의 문제 설명을 기반으로 판단한다.
